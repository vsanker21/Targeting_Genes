# Alignment with the GLIOMA-TARGET project outline

The authoritative narrative is **`Project_Outline.docx`** (this repo includes a text extract: **`Project_Outline_extracted.txt`**). The seven modules are **sequential and iterative** in the outline; this repository implements **early-stage computational scaffolding** toward that architecture.

Status summaries are also in **`config/pipeline_outline.yaml`**.

Snakemake optional **`rule all`** flags (**`GLIOMA_TARGET_INCLUDE_*`** — ClinVar, scRNA, MOVICS, supplementary wiring, M7 figures/docx): **[OPTIONAL_SLICES.md](OPTIONAL_SLICES.md)**. Selected automation drivers use **[snakemake_subprocess_env.py](../scripts/snakemake_subprocess_env.py)** so Snakemake subprocesses see the **default** `rule all` even when your shell still exports those variables.

**Data layout:** Use the same **`data_root`** as the GBM Detection project (see **`config/data_sources.yaml`**, env **`GLIOMA_TARGET_DATA_ROOT`**). Bulk and open-source pulls: **`docs/GBM_DETECTION_AND_OPEN_DATA.md`**, **`scripts/download_all_required.py`**, optional **`--with-open-extensions`** / **`config/open_source_data_extensions.yaml`** (CTD, optional GEO series), optional **`--with-supplementary-reference-resources`** / **`config/supplementary_reference_resources.yaml`** (Expression Atlas slices, ARCHS4/recount HDF5, pathway GMTs, ChEMBL + DGIdb, ClinVar + gnomAD; optional DrugCentral); Snakemake **`supplementary_reference_resources_paths_status`** is part of **`rule all`**, and **`GLIOMA_TARGET_INCLUDE_SUPPLEMENTARY_WIRING=1`** adds WikiPathways URL check, PathwayCommons plain GMT, fgsea plan/R stub, and ARCHS4 HDF5 summary, optional **`--with-high-value-datasets`** / **`config/high_value_public_datasets.yaml`** (GDSC, PRISM, Open Targets, GDC methylation beta, CELLxGENE Census h5ad, CGGA hook), and LINCS **CLUE** when **`CLUE_API_KEY`** / **`CLUE_API_TOKEN_FILE`** is set or **`scripts/fetch_clue_api_subset.py`** is run.

**Aggregated path index:** **`results/pipeline_results_index.json`** merges **M3–M7** export manifests plus job-list provenance JSONs (STRING/GSEA, WGCNA, LINCS, M6 bridge, M7 GTS; deduplicated paths, fresh `exists` / size / mtime). Provenance fields may store **absolute** paths; the index normalizes them to **repo-relative POSIX** keys so they merge with manifest rows. Manifest rows may set `optional`; **`pipeline_inventory.yaml`** → **`optional_path_posix`** can mark more paths optional after merge. Summary includes **`index_schema`** (integer; bump when the summary shape changes), **`sources_summary`** (how many manifest/provenance JSONs were found), sorted **`missing_required_path_posix`** / **`missing_optional_path_posix`**, counts **`n_missing_*`**, **`n_optional`**, **`optional_path_posix_unmatched`**, and per-module breakdowns. Config: **`config/pipeline_inventory.yaml`**, rule **`pipeline_results_index`**, script **`scripts/write_pipeline_results_index.py`**. Strict gates: **`--strict`** or env **`GLIOMA_TARGET_PIPELINE_INDEX_STRICT`** (`1` / `true` / `yes` / `on`; **`0` / `false` / `no` / `off`** or unset turns env strict off). On success the JSON records **`strict_triggers`** as **`["cli"]`**, **`["env"]`**, or **`["cli","env"]`**.

## Deliberate methodology choices vs outline text

1. **Primary DEA uses the TOIL RSEM hub**, not GDC STAR tumors mixed with TOIL normals, to avoid **batch confounding** (pipeline + reference + normalization). This is **stricter** than naively running DESeq2 on mismatched quantifications. See **`docs/DEA_METHODOLOGY.md`**.

2. **Normal brain panel:** The outline mentions **GTEx frontal lobe and cerebellum**. The default **`config/dea_tumor_normal.yaml`** uses **cerebral cortex / frontal / anterior cingulate** and **excludes cerebellum** from the pooled normal control to avoid **region-driven false positives** for hemispheric GBM. To match the outline’s cerebellum-inclusive control explicitly, add the relevant **`gtex_brain_detailed_categories`** entries and document the biological trade-off.

3. **Outline §2.1 tool list (DESeq2, edgeR):** Appropriate when **integer counts** exist for **both** arms on a **shared** processing path. This repo **prepares** TCGA STAR counts (`tcga_gbm_star_unstranded_counts_matrix.parquet`) for **within-TCGA** or future **matched** normal counts; primary discovery remains the TOIL **log-TPM** screen until counts are harmonized.

4. **FDR & |log2FC|:** The outline specifies **FDR < 0.05** and **|log2FC| > 1.5** as DE criteria. On the TOIL hub scale, **mean log₂(TPM+pseudo) differences** are **not** identical to DESeq2 log₂ fold change but serve as a **screening analogue**. Columns **`outline_m21_*`** mark genes meeting the outline **numerical** gates on the chosen effect column; interpret with **`docs/DEA_METHODOLOGY.md`**.
